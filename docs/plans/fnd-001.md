## Goal
Stand up the empty pnpm monorepo skeleton that every later ticket builds on:
a workspace root, two empty workspace packages (`apps/web`, `services/api`),
baseline editor/VCS hygiene files, and a root README that documents how to
install, build, dev, and test. This is purely structural — no framework
dependencies, no source code, no tooling config beyond what the acceptance
criteria require. It unblocks the rest of epic 01 (FND-002 strict TS, FND-004
Neon/Drizzle, FND-005 Hono, FND-009 Vite), all of which depend on FND-001.

## Acceptance criteria (from ticket)
- `pnpm install` succeeds at the repo root
- `pnpm -r build` runs (even with empty stubs)
- Root README documents how to install, build, dev, test

## Approach

The repo currently contains only docs, `.claude/`, `CLAUDE.md`,
`process-definition.md`, and a one-line `README.md`. No `package.json`,
`pnpm-workspace.yaml`, `.gitignore`, or `.editorconfig` exist yet. Every file
below is a new file except `README.md`, which is rewritten.

1. Create `pnpm-workspace.yaml` declaring the workspace globs:
   ```yaml
   packages:
     - "apps/*"
     - "services/*"
   ```
   (Globs, not exact paths, so FND-009 etc. can add siblings without editing
   this file. Matches the spec's `apps/` + `services/` layout and leaves room
   for a future `packages/` promotion noted in the spec.)

2. Create the root `package.json`:
   - `"name": "picksleagues"`, `"private": true`, `"type": "module"`
   - `"packageManager": "pnpm@<pinned version>"` (see Assumptions)
   - `"engines": { "node": ">=20" }` (see Assumptions)
   - Scripts that fan out to workspaces via `pnpm -r`:
     - `"build": "pnpm -r build"`
     - `"dev": "pnpm -r --parallel dev"`
     - `"test": "pnpm -r test"`
     - `"lint": "pnpm -r lint"`
     - `"typecheck": "pnpm -r typecheck"`
   - No dependencies or devDependencies. Tooling (TypeScript, ESLint,
     Prettier) is explicitly deferred to FND-002/FND-003.
   - Rationale for adding `dev`/`lint`/`typecheck` now even though the ticket
     only names `build`: the root README must document the dev/test workflow
     (AC #3), and CLAUDE.md's pre-commit hook calls `pnpm lint && pnpm
     typecheck`. Defining the passthrough scripts now keeps those commands
     resolvable; the per-package stubs (step 4) make them no-op-safe.

3. Create `.nvmrc` at the repo root pinning the Node major version (see
   Assumptions), so contributors and CI agree on the runtime.

4. Create the two workspace packages, each with a minimal `package.json` only
   (no `src/`, no stub `.ts` files — code-standards.md forbids "empty stub
   files 'for later'"; a directory with just a `package.json` is acceptable
   scaffolding, an empty `index.ts` is not):
   - `apps/web/package.json`:
     - `"name": "@picksleagues/web"`, `"private": true`, `"version": "0.0.0"`,
       `"type": "module"`
     - Scripts: `build`, `dev`, `test`, `lint`, `typecheck` — each a no-op
       placeholder that exits 0 (see Assumptions for the exact stub command).
   - `services/api/package.json`: same shape, name `@picksleagues/api`.
   These satisfy `pnpm -r build` (and the other fan-out scripts) running
   cleanly against empty packages. Later tickets replace each script body with
   the real command (Vite build, tsc, vitest, etc.).

5. Create `.gitignore` covering Node/pnpm/editor/OS artifacts that the rest of
   the project will generate:
   - `node_modules/`
   - `dist/`, `build/`, `.output/`
   - `.env`, `.env.*` (but not `.env.example`)
   - `*.log`, `pnpm-debug.log*`
   - `.DS_Store`
   - `.vercel/`
   - coverage output (`coverage/`)
   - TanStack Router generated file (`apps/web/src/routeTree.gen.ts`) is **out
     of scope** here — added by FND-009 when that file exists.

6. Create `.editorconfig` with baseline settings consistent with
   code-standards.md (kebab-case files, Prettier defaults — so 2-space indent,
   LF line endings, UTF-8, final newline, trim trailing whitespace; Markdown
   exempted from trailing-whitespace trim to preserve hard line breaks).

7. Rewrite root `README.md` (currently just `# Picks-Leagues`) to include:
   - Project one-liner + link to `docs/product-vision.md`
   - Prerequisites: Node version (from `.nvmrc`), pnpm via `corepack enable`
   - Repo layout: `apps/web`, `services/api`, `docs/`
   - Commands section documenting **install, build, dev, test** (the AC),
     i.e. `pnpm install`, `pnpm -r build` / `pnpm build`, `pnpm dev`,
     `pnpm test`
   - Links to the authoritative docs: MVP spec, game types, code standards,
     UI standards, backlog (mirrors the doc map in CLAUDE.md)
   - A short note that packages are intentionally empty stubs at this ticket
     and get fleshed out by subsequent FND tickets

## Risks and open questions
- The pre-commit hook (`.claude/hooks/pre-commit-check.sh`) runs
  `pnpm lint && pnpm typecheck`. With no ESLint/TS installed yet, these must
  resolve to a clean exit. The no-op stub scripts (step 4) handle this; the
  hook is also documented as "skipped gracefully if tooling isn't installed
  yet" in CLAUDE.md, so either path is safe. The implementer should confirm
  the chosen stub command exits 0 on the target shell.
- pnpm version / Node version pins were verified against current Vercel
  support on 2026-05-17 (Node 24 GA LTS / pnpm 11.1.2). One-line change in
  `package.json` / `.nvmrc` if a future bump is needed.

## Assumptions (autonomous mode only)
- **pnpm version:** pin `packageManager` to `pnpm@11.1.2` (latest stable,
  verified 2026-05-17). Requires Node `>=22.13`, satisfied by the Node 24 pin.
- **Node version:** `.nvmrc` = `24` and `engines.node` = `>=24`. Node 24 is
  the current Vercel-supported GA LTS and the default for new Vercel projects
  (verified 2026-05-17); it is the spec's serverless runtime target.
- **Workspace globs:** `apps/*` and `services/*` (glob form) rather than the
  two literal paths, so future packages need no edit here.
- **Stub script command:** each package script is
  `"echo \"<pkg>: no <task> step yet\" && exit 0"` (POSIX-portable, exits 0).
  An alternative is `"true"`, but the echo gives readable `pnpm -r` output.
- **Package names:** scoped `@picksleagues/web` and `@picksleagues/api` for
  unambiguous workspace references when later tickets add cross-package deps
  (e.g., the generated API client). Root package name `picksleagues`.
- **Extra root scripts:** `dev`, `test`, `lint`, `typecheck` added alongside
  the required `build`, because the README must document them (AC #3) and the
  pre-commit hook invokes `lint`/`typecheck`. They are thin `pnpm -r`
  passthroughs.
